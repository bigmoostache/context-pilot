Privacy Controls

Every aspect of Context Pilot is designed so you don't have to trust us — the architecture makes data exfiltration structurally impossible.

✓

No data collection

We never receive your code, conversations, memories, or any usage data. There is no server for it to go to.

✓

Local storage only

All state — conversations, entity databases, memories, logs, scratchpad — is stored on your filesystem under .context-pilot/. Nothing leaves the machine.

✓

API key isolation

Your API keys are stored in your local .env file. They are sent only to the provider they belong to — never to us or any third party.

✓

Open-source audit trail

The entire codebase is MIT-licensed on GitHub. Every network call, every file access, every tool invocation is auditable by you.

✓

Offline capable

Context Pilot works without internet. The only features requiring network are LLM inference, web search (Brave), and web scraping (Firecrawl) — all user-initiated and user-configured.

✓

Cryptographic integrity

Critical configuration files (lint rules, CI scripts, deploy scripts) are protected by a SHA-256 hash chain. Tampering is detected automatically.

✓

No vendor lock-in

MIT License. Swap LLM providers freely. Export your data anytime — it's already plain files on your disk.

Data Handling

Here's exactly what data goes where. Every item on this list is either stored locally or sent to a provider you explicitly configure.

⟠

Code context & conversations

Sent to your configured LLM provider (Anthropic, OpenAI, etc.) as part of inference requests. Governed by their data retention policies, not ours.

✓

Conversation history

Stored locally in .context-pilot/ as YAML files. Never transmitted anywhere.

✓

Entity database

Local SQLite database under .context-pilot/shared/entities/. Never transmitted.

✓

Memories, logs, scratchpad

Plain YAML/JSON files on your filesystem. Persist across sessions, never leave the machine.

✓

Search index (Meilisearch)

Runs as a local sidecar process on 127.0.0.1. Index data never leaves localhost.

Compliance

As a local-first, open-source tool with no cloud component, traditional compliance frameworks don't apply in the usual way. Here's our stance:

✓ Compatible GDPR

Context Pilot processes no personal data on our behalf. Any data sent to LLM providers is governed by your agreement with that provider. We are not a data processor — we're a tool you run locally.

N/A SOC 2

SOC 2 applies to cloud services that handle customer data. Context Pilot has no cloud service — your data never enters our control. SOC 2 certification would certify an empty infrastructure.

✓ MIT Open Source License

Permissive MIT license. Use commercially, modify, distribute. No vendor lock-in, no license fees, no usage restrictions.
