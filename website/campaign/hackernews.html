Handling Comments

If someone asks "how is this different from Cursor/Copilot/Aider?"
The key difference is context visibility. In Cursor/Copilot, the context window is a black box ‚Äî the tool decides what the AI sees, and neither you nor the AI know what's in it. In Context Pilot, every piece of context is a visible panel with a token count. The AI can see its own sidebar, know it's at 8K/200K tokens, decide that a file it opened 10 minutes ago is no longer relevant, and close it. It's the difference between a developer with a messy desk who can't find anything, and one who actively organizes their workspace. Aider is closer in spirit (terminal-based, direct), but it still treats context as implicit. You add files to the chat, and they stay there. Context Pilot's panels are dynamic ‚Äî the AI creates and destroys them as it works.
If someone asks "why not just use a bigger context window?"
Bigger windows help, but they don't solve the core problem. A 1M token window with 500K of stale context is worse than a 200K window where every token is relevant. The issue isn't capacity ‚Äî it's curation. When the AI can see its own context, it naturally curates. It closes files it's done with, archives old conversation chunks, takes notes instead of keeping raw data. The result is that it uses context more efficiently, not just more of it.
If someone asks about the AGPL license
AGPL because I want contributions to come back to the project. If you're using it as a local dev tool (which is the intended use case), AGPL doesn't affect you at all ‚Äî it only triggers if you're distributing or running it as a service. For commercial/enterprise use cases where AGPL is a concern, I'm open to dual licensing. Open an issue and we can talk.
